# 项目介绍

# 云康复App/小程序

一个专为科室以及医生、治疗师和患者及其家属的连接平台，为患者提供离院后训练指导等康复服务。

  • 使用uniapp框架，基于vue开发，兼容H5/APP/小程序等多端跨平台应用。
  • 使用vue-cli快速搭建开发环境，使用flyio配置拦截器以及网络请求，使用vuex和localstorage处理各组件间的通讯。
  • 使用promise来处理ajax回调函数，解决回调地狱问题。
  • 使用less预处理器，封装可复用的css样式。
  • 使用unaipp的downloadFile和html5+的install和restart等相关API实现app的热更新和检查更新功能。
  • 患者列表页面以卡片的形式展示，栏目分别有住院、门诊、会诊。使用unaipp的onPullDownRefresh和onReachBottom两个生命周期函数实现数据的下拉刷新和上拉加载功能。

# 云康复后台管理系统

一个专为科室以及医生、治疗师和患者及其家属的连接平台，为患者提供离院后训练指导等康复服务。

  • 基于element-ui框架，使用es6+typescript进行开发。
  • 使用vue-cli快速搭建开发环境，使用flyio配置拦截器以及网络请求，使用vuex和localstorage处理各组件间的通讯。
  • 使用promise来处理ajax回调函数，解决回调地狱问题。
  • 使用scss预处理器，封装可复用的css样式。
  • 使用html5的drag和drop属性实现拖放排班功能。在待分配列表中把患者拖放到表格进行排班，并且表格对应的格子之间也可以进行拖放排班（需要判断是否拖放到正确的格子(已安排、未安排、未确认、占用)以及当前表格格子时间段是否空闲）。
  • 新增子项目套餐时，可添加多个套餐项目并且填写对应套餐价格、部位数；同时，可添加多个工作日期对应多个时间段，表单提交时时间参数需要转换为今年的第几天和第几分钟的格式。
  • 患者详情页面中，划分为患者信息、住院信息、医嘱信息、治疗医嘱、已删医嘱、治疗安排、治疗记录等模块。在治疗记录模块中，只有该医嘱为未执行状态并且是系统当天同时还需要是治疗师或治疗师长身份下才能执行表单操作。表单中默认选中展示全部的套餐项目，并且可以对套餐项目的其他信息进行修改。如果该医嘱没有执行时间，则执行开始时间显示为当前时间，而执行结束时间显示为当前时间+治疗时长。

# 农业物联网综合平台

一个专为广大农业生产者提供可参考的农业信息化管理平台，帮助学习先进的农业知识，掌握科学农业的生产方法。

  • 基于element-ui框架，使用vue+es6+axios进行开发。
  • 使用vue-cli快速搭建开发环境，使用axios进行网络请求，使用vuex和localstorage处理各组件间的通讯。
  • 使用scss预处理器，封装可复用的css样式。
  • 首页划分为设备统计、警告分析、任务进度、实时警告等模块。在警告模块中使用vue-echarts图表的形式展示，可根据时间进行查询。
  • 角色管理页面中，可以自由设置用户页面权限和区域权限。在增加用户角色表单中，使用Tree树形控件展示页面权限、区域权限信息，根据用户选择进行表单提交。
  • 预览

# 膳食订餐钉钉小程序

以钉钉小程序为平台的膳食订餐系统，企业员工可通过该应用提供隔天订餐等便捷服务。

  • 使用uniapp框架，基于vue开发，兼容小程序/APP/H5等多端跨平台应用。
  • 使用vue-cli快速搭建开发环境，使用钉钉的httpRequest()配置拦截器以及网络请求，使用vuex和localstorage处理各组件间的通讯。
  • 使用promise来处理ajax回调函数，解决回调地狱问题。
  • 使用scss预处理器，封装可复用的css样式。
  • 使用钉钉的免登机制自动获取用户身份，使用递归实现token的刷新。
  • 点餐页面使用scroll-view实现左右菜单联动，每类商品区域通过高度计算实现滚动时菜单栏高亮显示，使用vuex实现购物车数据的管理。
  • 订单备注页面使用vuex（当前输入）和localstorage（搜索列表）保存数据到本地。
  • 订单列表使用钉钉的onPullDownRefresh和onReachBottom两个生命周期函数实现数据的下拉刷新和上拉加载功能。
  • 订单列表使用vuex解决每次进入页面都会触发onShow生命周期函数，只有结算、取消、删除订单以及餐卡解绑、挂失时才会刷新订单列表并且滚动到顶部的位置。
  • 余额充值页面提交充值金额得到订单信息，再使用钉钉的pay()支付接口实现支付宝充值功能。

# WebStack.cc

一个网址收藏夹，适用于轻量级网站。

  • 前端使用vue全家桶+ts+es6+axios+less+element-ui
  • 使用axios配置拦截器以及统一返回错误，使用vuex和localstorage处理各组件间的通讯。
  • 使用less预处理器，封装可复用的css样式。
  • 使用vue-router路由守卫实现登录前后路由拦截。
  • 服务端使用node+koa2+mongodb
  • 使用jsonwebtoken（JWT）生成token以及使用passport加密用户密码。
  • 使用concurrently模块来实现前后端连载。
